ThumbsAI v1.0

AI Image Browser — Designed by Zero  |  Built by Jarvis

Introduction

ThumbsAI is a fast, offline AI image browser for Windows. It generates and caches thumbnails for large collections of AI-generated images, reads generation metadata embedded in PNG files (prompt, seed, model, sampler, CFG scale), and lets you browse, search, sort, edit, and organize your images without any cloud dependency.

Key capabilities:

  • Reads AI metadata from Automatic1111 / AUTOMATIC1111, ComfyUI, NovelAI, and InvokeAI PNG files
  • Persistent SQLite thumbnail cache — folders open instantly after first scan
  • Watched folders that auto-update on app start and on disk changes
  • Photoshop .8bf plugin pipeline for non-destructive filters
  • Similarity sort — visually sort images by perceptual similarity to a selected image
  • Video thumbnail support (requires FFmpeg)
  • ZIP archive browsing

Installation

Step 1 — Run INSTALL.bat

Double-click INSTALL.bat. It will:

  1. Check for Python 3.11+. If not found, attempt to install it via winget.
  2. Create a Python virtual environment in the .venv\ folder.
  3. Install all required packages (PySide6, Pillow, pillow-avif-plugin).
Note If the optional pillow-avif-plugin fails to install (some systems lack a C compiler), the installer automatically retries without it. AVIF files will simply not be supported.

Step 2 — Launch the app

Double-click run.bat or create a desktop shortcut to it. The app opens with no console window.

Updating

To update to a new version:

  1. Close ThumbsAI.
  2. Run BACKUP_DATA.bat to snapshot your settings and database.
  3. Copy the new version files over your existing installation folder, skipping the data\ folder.
  4. Run INSTALL.bat again to update Python packages if needed.
  5. Launch with run.bat.
Tip Your thumbnail database and settings are stored in data\. Never overwrite that folder when updating unless you intend to start fresh.

First Run

  1. Launch via run.bat.
  2. In the Folder Panel on the left, expand the drive tree and click a folder containing images.
  3. ThumbsAI will scan the folder in the background and display thumbnails as they are generated.
  4. Folders you want to keep current automatically — right-click them in the panel and choose Always Watch this Folder.

The first scan of a large folder can take several minutes. Subsequent opens are instant (cached).

Favorites

Favorites give you quick access to a curated set of folders without scrolling through the full filesystem tree.

  • Right-click any folder → ☆ Add to Favorites.
  • Enable the ★ Favorites checkbox in the panel header to switch to the flat favorites view.
  • In Favorites mode the tree is replaced by a simple list — no filesystem scanning, instant display.
  • The Favorites checkbox state is saved and restored on every launch.
  • If you frequently work with a small set of folders, keeping Favorites enabled on startup noticeably speeds up initial load time.

Watched Folders

Watched folders are scanned automatically in two situations:

  • On app start — every watched folder is queued for a scan when ThumbsAI launches.
  • On file changes — the OS filesystem watcher detects new or modified files and triggers a re-scan within seconds.
Tip Use watched folders for your active output directories (e.g. your A1111 outputs\ folder). New images will appear in the grid automatically without any manual action.
Note The file watcher monitors up to 120 subdirectory paths per watched root. Very deep directory trees may not have all leaf folders watched, but the root and immediate children are always covered.

Watched Folders vs Auto-Scan

FeatureAuto-ScanWatched Folders

Folders coveredWhichever folder you clickOnly folders on the watch list
Good forExploring all foldersActive output directories you always want current

You can enable both. They deduplicate — if you click a watched folder while it is already being scanned, a second scan is not started.

Scanning

How Scanning Works

When a folder is scanned, ThumbsAI:

  1. Walks the folder tree (including all sub-folders) using fast OS-level directory enumeration.
  2. Compares each file's modification time against the cached value in the database.
  3. Skips files whose mtime matches the cache — unchanged files cost almost nothing.
  4. For changed or new files, reads the image, generates a thumbnail, extracts AI metadata, and stores everything in the database.
  5. Emits each completed thumbnail to the grid in real-time — you see results as they appear, not only at the end.
Note Scanning runs on a background thread and never blocks the UI. You can browse other folders and use all other features while a scan is in progress.

AI Metadata — Overview

ThumbsAI reads AI generation metadata embedded in PNG files at scan time. No internet connection is needed — all parsing happens locally from the PNG tEXt chunk.

The following fields are extracted and stored in the database:

FieldDescription
PromptThe positive text prompt used to generate the image
Negative PromptThe negative prompt (things to exclude)
ModelCheckpoint model name or hash
SamplerSampling algorithm (Euler a, DPM++ 2M, etc.)
SeedRandom seed for reproducibility
StepsNumber of denoising steps
CFG ScaleClassifier-free guidance scale
TagsUser-defined tags (editable)
SourceWhich generator produced the file (A1111, ComfyUI, etc.)
Raw MetaThe complete original metadata string for reference

Viewing Metadata

There are two ways to open the metadata Properties dialog:

  • Right-click a thumbnail → Metadata
  • Open the image in the viewer → right-click → Metadata

The Properties dialog shows all stored metadata fields. If the image has not been scanned yet, the fields will be blank.

You can copy all metadata to the clipboard at once using the Copy All button — useful when pasting generation parameters into A1111 or CivitAI.

Drag-to-Website

When you drag a PNG from ThumbsAI onto a website (such as CivitAI's image upload), the original PNG file is transferred — including any embedded metadata. CivitAI and similar sites read the parameters chunk directly from the file, so your generation details will be recognized automatically, provided the metadata was originally written to the file (not just stored in the ThumbsAI database).

Tip If you edit metadata in ThumbsAI and click Save, the metadata is written back to the PNG file so drag-and-drop to external sites will always carry the current values.

Editing & Saving Metadata

In the Properties dialog, click Edit to unlock all fields. Make your changes, then click Save.

What Save does

  • Always: Writes all fields to the ThumbsAI database immediately.
  • For PNG files: Also rewrites the parameters tEXt chunk in the file using the A1111 format. The write is atomic — a temporary file is written first, then renamed, so a power failure cannot corrupt the original.
  • For JPEG, WebP, and other formats: Metadata is saved to the database only. A notice is shown confirming this. The file on disk is not modified.
Note Only the parameters chunk is modified. Pixel data, ICC colour profile, and all non-AI text chunks (e.g. software, creation_time) are preserved exactly as-is.

Format Written

Metadata is saved in A1111 format, which is the most widely recognised across AI tools:

your positive prompt here
Negative prompt: your negative prompt here
Steps: 30, Sampler: Euler a, CFG scale: 7, Seed: 1234567890, Model: v1-5-pruned

This format is understood by A1111, CivitAI, ComfyUI image loaders, and most other AI image tools.

Supported Generator Formats

GeneratorDetection KeyNotes
Automatic1111 / AUTOMATIC1111parameters tEXt chunkFully supported. Steps, sampler, CFG, seed, model, negative prompt all parsed.
ComfyUIprompt tEXt chunk (JSON node graph)Extracts CLIPTextEncode nodes for prompts, KSampler for seed/steps/CFG, CheckpointLoader for model name.
NovelAIDescription and/or Comment chunksPrompt from Description; seed, sampler, steps, scale from Comment JSON.
InvokeAIsd-metadata tEXt chunkPrompt, seed, steps, CFG, sampler, model from JSON metadata blob.

Files from other tools without recognised metadata chunks are still indexed and displayed — the metadata fields will simply be empty.

Photoshop .8bf Plugins

ThumbsAI can apply Photoshop-compatible .8bf filter plugins directly to images. These are the same plugins that run in Photoshop, Affinity Photo, and similar hosts — no Photoshop installation required.

Setup

  1. Open Settings (Ctrl+,).
  2. Under Photoshop Plugin Directories, add the folder(s) where your .8bf files are stored.
  3. ThumbsAI will scan those directories and populate the Apply Filter submenu.

Plugins are organised by category in the right-click menu. A Redfield plugin collection is included with ThumbsAI in the plugins\Redfield\ folder.

Included Redfield Plugins

Fractalius, Fractalius 2, FaceControl, FineTouch, Perfectum 2, Sharpinion, SharpinionHD, SketchMaster, iEmboss, UMATRIX, and Atx.

Launch Apps

Configure external applications to open images from the right-click menu under Open With.

  1. Open Settings (Ctrl+,) → Launch Apps.
  2. Add an entry: name, path to the executable, and optional arguments. Use {file} as a placeholder for the image path.

Common uses: open in Photoshop, load into A1111 img2img, send to your upscaler, etc.

Your Data

All user-specific data lives in the data\ subfolder of the ThumbsAI installation directory. Never overwrite this folder when updating.

File Locations

FileContentsSize

data\settings.jsonAll application settings including favorites list, watched folders list, launch apps, sort state, font sizes, and moreSmall (a few KB)

Watch List & Favorites are stored inside settings.json under the keys "watched_folders" and "favorites". There is no separate watch-list file — back up settings.json to preserve both.

Backup & Restore

Use the provided batch scripts:

ScriptPurpose
BACKUP_DATA.batCreates a timestamped snapshot in _backups\YYYY-MM-DD_HHmmss\. Always backs up settings.json; optionally backs up thumbs.db (can be large).
RESTORE_DATA.batLists available snapshots and restores selected files back to data\. Run with ThumbsAI closed.

Regenerating the Thumbnail Database

If you delete thumbs.db or start fresh, ThumbsAI will recreate it automatically on the next launch. All previously scanned folders will appear yellow (not scanned) and will need to be re-scanned. Your settings.json (favorites, watched folders, etc.) is unaffected.

Troubleshooting

App takes a long time to open

The first launch after a system restart may take 10–30 seconds if a large database is present — SQLite performs background optimisation at startup. Subsequent launches are faster. If the app takes more than 2 minutes, check data\startup_error.log for errors.

App says "ThumbsAI is already running"

ThumbsAI uses a lock file (~\.thumbsai.lock in your home folder) to prevent duplicate instances. If the app crashed previously, the stale lock can block the next start. The lock automatically expires after 10 seconds. If the message persists, delete %USERPROFILE%\.thumbsai.lock manually.

Folder colors are all yellow after opening

Folder colors are populated from the database in a background thread shortly after startup. Wait a few seconds, or click the ⟳ button in the folder panel header to force a refresh.

Thumbnails are not generating

  • Check that the folder is not in the Disabled Extensions settings list.
  • Check the Task Queue for errors — right-click → Show Errors.
  • For videos, ensure FFmpeg is installed and its path is set in Settings.

Metadata is empty for a PNG that was generated with A1111

  • Some batch or post-processing tools strip PNG metadata. Re-open the PNG in an image editor and check if tEXt chunks are present.
  • In A1111, ensure Save text information about generation parameters as chunks to png files is enabled in Settings → Saving images.

Metadata was edited but the file still shows old values when dragged to a website

After editing in the Properties dialog, you must click Save (not just close). Save writes the updated metadata both to the database and to the PNG file on disk. Metadata stored only in the database is not embedded in the file.

AVIF files are not showing

AVIF support requires pillow-avif-plugin. Re-run INSTALL.bat to attempt installation. If it fails (usually due to a missing system C compiler), AVIF will not be available on that machine.

Crash on startup — blank window or immediate close

Check data\startup_error.log for the full traceback. Common causes:

  • Python version mismatch — ensure Python 3.11+ is installed and used by the .venv.
  • Corrupted virtual environment — delete the .venv\ folder and re-run INSTALL.bat.
  • Missing PySide6 — re-run INSTALL.bat.

Contacting Support

ThumbsAI is designed and maintained privately. Include the contents of data\startup_error.log with any bug report.
